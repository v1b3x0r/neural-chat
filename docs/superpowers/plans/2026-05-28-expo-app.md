# Living-Memory Chat — Expo App (Plan 2)

> Build plan for the Expo app on top of the proven `@nature-labs/living-memory-engine`.
> **Verification note:** unlike Plan 1 (headless TDD), most of this is UI that must be verified on-device via Expo Go. Code is written to current **Expo SDK 56** APIs (verified against docs.expo.dev/versions/v56.0.0). Each phase ends with a device-verify step the human runs.

## Phase A — SQLite StoragePort

**Files:** Create `mobile/src/lib/sqlite-storage.ts`

Implements the engine's `StoragePort` (load/save `Snapshot`) on expo-sqlite. One DB file per persona namespace (the P1 seam — pass `namespace`, default `'default'`). Whole-snapshot load/save is fine for a personal app's data volume; optimize later only if needed.

```ts
import * as SQLite from 'expo-sqlite';
import type { StoragePort, Snapshot } from '@nature-labs/living-memory-engine';

const DDL = `
  CREATE TABLE IF NOT EXISTS kv (k TEXT PRIMARY KEY, v TEXT NOT NULL);
`;

export async function makeSqliteStorage(namespace = 'default'): Promise<StoragePort> {
  const db = await SQLite.openDatabaseAsync(`neural-chat-${namespace}.db`);
  await db.execAsync(DDL);

  const empty: Snapshot = { messages: [], episodic: [], selfFacets: [], prospective: [], lastTick: 0 };

  return {
    async load() {
      const row = await db.getFirstAsync<{ v: string }>('SELECT v FROM kv WHERE k = ?', ['snapshot']);
      return row ? (JSON.parse(row.v) as Snapshot) : empty;
    },
    async save(s: Snapshot) {
      await db.runAsync(
        'INSERT INTO kv (k, v) VALUES (?, ?) ON CONFLICT(k) DO UPDATE SET v = excluded.v',
        ['snapshot', JSON.stringify(s)],
      );
    },
  };
}
```

**Device-verify:** temporary button calls `save` then `load`, log round-trips a snapshot. Remove after.

> Note: whole-snapshot JSON keeps embeddings in the DB. If the DB grows heavy later, split episodic into rows with embeddings as BLOB — defer (YAGNI).

## Phase B — Provider config + secure key

**Files:** Create `mobile/src/lib/config.ts`, `mobile/src/app/settings.tsx`

- Endpoints (chat = OpenRouter, embed = LM Studio) stored via `localStorage` polyfill (`expo-sqlite/localStorage/install`); **API key stored only in `expo-secure-store`** (`setItemAsync`/`getItemAsync`), never in plain storage, never bundled.
- `settings.tsx`: minimal form — chat baseURL/model/key, embed baseURL/model. Hidden behind the drawer, not on the chat screen.

```ts
// config.ts (shape)
import * as SecureStore from 'expo-secure-store';
import 'expo-sqlite/localStorage/install';
export interface EndpointCfg { baseURL: string; model: string; }
export async function getChatKey() { return (await SecureStore.getItemAsync('openrouter_key')) ?? ''; }
export async function setChatKey(k: string) { await SecureStore.setItemAsync('openrouter_key', k); }
export function getChatCfg(): EndpointCfg { return JSON.parse(localStorage.getItem('chatCfg') ?? '{"baseURL":"https://openrouter.ai/api/v1","model":""}'); }
export function getEmbedCfg(): EndpointCfg { return JSON.parse(localStorage.getItem('embedCfg') ?? '{"baseURL":"http://localhost:1234/v1","model":"nomic-embed-text"}'); }
// setters mirror getters via localStorage.setItem
```

**Device-verify:** enter key in settings, reopen app, confirm it persists (secure-store survives reload).

## Phase C — Engine wiring (per-persona instance)

**Files:** Create `mobile/src/lib/engine-instance.ts`

Builds a `MemoryEngine` for a namespace: SQLite storage + provider adapters (chat/embed) + real `Clock` (`{ now: Date.now }`) + `SeededRandom(seed)` (seed persisted per persona) + `randomK(3,7)` default policy (swappable from debug).

```ts
import { MemoryEngine, SeededRandom, randomK, fixedK, gamble } from '@nature-labs/living-memory-engine';
import { makeChatPort, makeEmbedPort } from '@nature-labs/living-memory-engine/provider';
import { makeSqliteStorage } from './sqlite-storage';
import { getChatCfg, getEmbedCfg, getChatKey } from './config';

export async function makeEngine(namespace = 'default', systemPrompt = '') {
  const storage = await makeSqliteStorage(namespace);
  const chatCfg = { ...getChatCfg(), apiKey: await getChatKey() };
  const embedCfg = { ...getEmbedCfg(), apiKey: '' };
  return new MemoryEngine({
    storage,
    chat: makeChatPort(chatCfg),
    embed: makeEmbedPort(embedCfg),
    clock: { now: () => Date.now() },
    random: new SeededRandom(/* persisted seed */ 1337),
    policy: randomK(3, 7),
    systemPrompt,
  });
}
```

**Device-verify:** instantiate engine on app load, call `retrieve('hi')` once, log the empty injection context without crashing.

## Phase D — Chat screen (single thread)

**Files:** `mobile/src/app/index.tsx` (the one thread), `mobile/src/components/message-bubble.tsx`, `mobile/src/components/composer.tsx`

- Inverted `FlatList` of messages, `contentInsetAdjustmentBehavior="automatic"`, empty state = logo + greeting (Gemini-style).
- Composer: text field + `＋` (image) + send. **No "new chat" anywhere.**
- Send flow: `ingestUser(text)` → `retrieve(text)` → build inject string from `InjectionContext` → `chat.stream(tail, systemPrompt, inject)` render streaming → on done, `tick()` (debounced shallow); deep `tick()` also on app foreground.
- Links from model output: render only if scheme `^https?://` + `rel`-equivalent (open via `expo-web-browser`), per spec §5.1.

**Device-verify:** type a few messages over a "session", confirm replies stream and the thread persists across app reloads (SQLite). Confirm no full-history is sent (inject is bounded).
